# 置信度优化策略

## 🎯 问题分析

你的观察非常准确！**带路径参数的规则置信度确实偏低**，这会导致：

1. 更精确的规则（带参数）被通用规则（无参数）抢先匹配
2. 生成的命令缺少必要的参数
3. 命令执行失败或行为不符合预期

## 📊 当前置信度分布

### 文件和目录操作

| 规则类型 | 当前置信度 | 问题 | 建议置信度 |
|---------|-----------|------|-----------|
| **盘符查询**（无路径提取）| 0.95 | ✅ 合理 | 0.95 |
| **特殊处理**（SPECIAL:create_folders）| 0.92-0.95 | ✅ 合理 | 0.95 |
| **特殊处理**（SPECIAL:delete_folder）| 0.92 | ⚠️ 应该更高 | **0.95** |
| **切换目录**（带路径）| 0.90 | ⚠️ 偏低 | **0.93** |
| **创建文件夹**（带名称）| 0.90 | ✅ 合理 | 0.90 |
| **删除文件**（带路径）| 0.85 | ❌ 太低 | **0.88** |
| **创建文件夹**（无参数）| 0.85 | ❌ 太高 | **0.75** |
| **删除文件夹**（无参数）| 0.80 | ⚠️ 应该更低 | **0.70** |

## 🎨 优化原则

### 1. 参数越多，置信度越高

```
特殊处理 + 多参数  → 0.95  (最高)
特殊处理 + 单参数  → 0.93
普通规则 + 路径    → 0.88-0.93
普通规则 + 名称    → 0.90
普通规则 + 无参数  → 0.70-0.75  (兜底)
```

### 2. 特殊处理优先

```python
# 特殊处理规则（SPECIAL:）应该有最高置信度
r'删除(.+?)(的)?([^\s]+)(文件夹|目录)': (
    'SPECIAL:delete_folder_at_location',
    '删除指定位置的文件夹',
    0.95  # ✅ 最高：特殊处理 + 位置 + 名称
)

# 带路径的通用规则次之
r'(删除|移除|rm).*(文件|目录).*?([^\s]+)': (
    'Remove-Item {path}',
    '删除指定文件或目录',
    0.88  # ✅ 较高：带路径参数
)

# 无参数规则最低（兜底）
r'(删除|移除|rm).*文件': (
    'Remove-Item',
    '删除文件',
    0.70  # ✅ 低：无参数，仅作兜底
)
```

### 3. 精确度决定置信度

| 精确度 | 置信度范围 | 示例 |
|--------|-----------|------|
| **非常精确** | 0.95-0.98 | 特殊处理 + 多参数 |
| **精确** | 0.90-0.93 | 带路径/名称参数 |
| **一般** | 0.85-0.88 | 带单个参数 |
| **模糊** | 0.70-0.80 | 无参数（兜底） |

### 方案2: 动态调整

在代码中添加置信度调整逻辑：

```python
def _adjust_confidence(self, base_confidence: float, has_params: bool, 
                       is_special: bool, param_count: int) -> float:
    """动态调整置信度"""
    confidence = base_confidence
    
    # 特殊处理加分
    if is_special:
        confidence += 0.05
    
    # 参数加分
    if has_params:
        confidence += 0.03 * param_count
    
    # 限制在合理范围
    return min(0.98, max(0.70, confidence))
```
