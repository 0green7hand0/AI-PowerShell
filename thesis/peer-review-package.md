## 三、审阅指南

### 3.1 第一遍：快速浏览

**目标**: 了解整体结构和主要内容

**方法**:
1. 阅读摘要和目录
2. 浏览各章节标题
3. 看看图表和代码示例
4. 形成整体印象

**记录**:
- 整体感觉如何？
- 结构是否清晰？
- 有哪些亮点？
- 有哪些疑问？

### 3.2 第二遍：重点细读

**目标**: 深入理解核心内容

**重点章节**:
1. **第1章 绪论** - 研究背景和意义
2. **第3章 需求分析** - 系统要做什么
3. **第4章 总体设计** - 系统怎么设计
4. **第6章 测试分析** - 系统效果如何

**阅读技巧**:
- 遇到不理解的地方做标记
- 发现错误或不合理之处记录下来
- 思考如果是你会怎么写

### 3.3 第三遍：格式检查

**目标**: 发现格式和表达问题

**检查要点**:
- [ ] 图表编号是否连续
- [ ] 图表是否有标题
- [ ] 参考文献格式是否统一
- [ ] 是否有明显的错别字
- [ ] 标点符号是否正确
- [ ] 代码格式是否规范

## 附录: 审阅技巧

### A1. 如何快速发现问题

**看标题**:
- 标题是否清晰？
- 标题层次是否合理？
- 标题是否概括内容？

**看图表**:
- 图表是否清晰？
- 图表编号是否连续？
- 图表是否有标题和说明？

**看数据**:
- 数据是否一致？
- 数据是否合理？
- 数据是否有来源？

**看逻辑**:
- 前后是否一致？
- 论证是否充分？
- 结论是否合理？

### A2. 如何提出建设性意见

**好的反馈**:
- ✅ 具体: "第3章第2节的图3-2不清晰"
- ✅ 建设性: "建议增加对XX的说明"
- ✅ 客观: "这里的逻辑不太清楚"

**不好的反馈**:
- ❌ 笼统: "写得不好"
- ❌ 主观: "我不喜欢这种写法"
- ❌ 打击: "这部分完全不行"

### A3. 常见问题检查清单

**内容方面**:
- [ ] 研究背景是否充分？
- [ ] 研究意义是否明确？
- [ ] 技术方案是否合理？
- [ ] 实验数据是否充分？
- [ ] 结论是否有理有据？

**格式方面**:
- [ ] 图表编号是否连续？
- [ ] 参考文献格式是否统一？
- [ ] 标点符号是否正确？
- [ ] 是否有错别字？

**语言方面**:
- [ ] 表达是否清晰？
- [ ] 逻辑是否连贯？
- [ ] 术语使用是否准确？
- [ ] 是否有歧义？
