# 毕业论文格式规范化指南

## 文档说明

本指南用于统一AI PowerShell智能助手毕业设计的所有文档格式，确保文档的专业性、规范性和一致性。

**适用范围：**
- 毕业论文主文档
- 系统设计文档
- 用户手册
- 开发文档
- 测试报告
- 答辩材料

**版本信息：**
- 版本：v1.0
- 创建日期：2024年
- 最后更新：2024年

## 1. 页面设置规范

### 1.1 纸张和页边距

**纸张规格：**
- 纸张大小：A4 (210mm × 297mm)
- 纸张方向：纵向

**页边距设置：**
- 上边距：2.54cm (1英寸)
- 下边距：2.54cm (1英寸)
- 左边距：3.17cm (1.25英寸)
- 右边距：3.17cm (1.25英寸)
- 装订线：0cm
- 装订线位置：左侧

### 1.2 行距和段落

**行距设置：**
- 正文行距：1.5倍行距
- 标题行距：1.5倍行距
- 段前间距：0.5行
- 段后间距：0.5行

**段落设置：**
- 首行缩进：2字符（中文段落）
- 对齐方式：两端对齐
- 孤行控制：启用
- 寡行控制：启用

### 1.3 页眉页脚

**页眉设置：**
- 内容：基于本地AI模型的智能PowerShell命令行助手设计与实现
- 字体：宋体，五号
- 对齐：居中
- 距边界：1.5cm

**页脚设置：**
- 内容：页码
- 格式：第X页 共Y页 或 - X -
- 字体：Times New Roman，五号
- 对齐：居中
- 距边界：1.75cm

## 2. 字体和字号规范

### 2.1 中文字体

**标题字体：**
- 一级标题（章）：黑体，三号（16pt），加粗，居中
- 二级标题（节）：黑体，四号（14pt），加粗，左对齐
- 三级标题（小节）：黑体，小四号（12pt），加粗，左对齐
- 四级标题：黑体，小四号（12pt），左对齐

**正文字体：**
- 正文内容：宋体，小四号（12pt）
- 图表标题：宋体，五号（10.5pt），居中
- 图表内容：宋体，五号（10.5pt）
- 页眉页脚：宋体，五号（10.5pt）
- 脚注：宋体，小五号（9pt）

### 2.2 英文和数字字体

**标题字体：**
- 一级标题：Times New Roman，三号（16pt），加粗，居中
- 二级标题：Times New Roman，四号（14pt），加粗，左对齐
- 三级标题：Times New Roman，小四号（12pt），加粗，左对齐

**正文字体：**
- 正文内容：Times New Roman，小四号（12pt）
- 图表标题：Times New Roman，五号（10.5pt）
- 图表内容：Times New Roman，五号（10.5pt）

### 2.3 代码字体

**代码块字体：**
- 字体：Consolas 或 Courier New
- 字号：小五号（9pt）
- 行号：显示
- 背景：浅灰色（#F5F5F5）
- 边框：1pt实线，灰色

**行内代码字体：**
- 字体：Consolas 或 Courier New
- 字号：与正文相同
- 背景：浅灰色（#F0F0F0）
- 边框：无

## 3. 标题格式规范

### 3.1 标题编号

**章节编号格式：**
- 第一级（章）：第X章
- 第二级（节）：X.Y
- 第三级（小节）：X.Y.Z
- 第四级：X.Y.Z.W

**示例：**
```
第1章 绪论
1.1 研究背景与意义
1.1.1 研究背景
1.1.1.1 命令行工具的重要性
```

### 3.2 标题样式

**第一级标题（章）：**
- 格式：第X章 标题内容
- 字体：黑体，三号，加粗
- 对齐：居中
- 段前：24pt
- 段后：18pt
- 另起一页：是

**第二级标题（节）：**
- 格式：X.Y 标题内容
- 字体：黑体，四号，加粗
- 对齐：左对齐
- 段前：18pt
- 段后：12pt
- 另起一页：否

**第三级标题（小节）：**
- 格式：X.Y.Z 标题内容
- 字体：黑体，小四号，加粗
- 对齐：左对齐
- 段前：12pt
- 段后：6pt
- 另起一页：否

## 5. 参考文献格式规范（GB/T 7714-2015）

### 5.1 文献类型标识

**文献类型代码：**
- [M] - 专著（Monograph）
- [J] - 期刊文章（Journal）
- [D] - 学位论文（Dissertation）
- [C] - 论文集（Collection）
- [N] - 报纸文章（Newspaper）
- [R] - 报告（Report）
- [S] - 标准（Standard）
- [P] - 专利（Patent）
- [EB/OL] - 网络资源（Electronic/Online）
- [DB/OL] - 数据库（Database/Online）
- [CP/DK] - 计算机程序（Computer Program/Disk）

### 5.2 期刊文章格式

**基本格式：**
```
[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
```

**示例：**
```
[1] 张三, 李四. 基于深度学习的自然语言处理研究[J]. 计算机学报, 2023, 46(5): 1023-1035.
[2] Smith J, Johnson M. Natural language processing with transformers[J]. IEEE Transactions on Neural Networks, 2023, 34(2): 256-270.
```

**注意事项：**
- 作者≤3人：全部列出
- 作者>3人：列出前3人，后加"等"或"et al"
- 中文作者用逗号分隔
- 英文作者用", "分隔

### 5.3 专著格式

**基本格式：**
```
[序号] 作者. 书名[M]. 版本. 出版地: 出版者, 年: 起止页码.
```

**示例：**
```
[3] 王五. Python编程从入门到实践[M]. 第2版. 北京: 人民邮电出版社, 2022: 45-67.
[4] Russell S, Norvig P. Artificial Intelligence: A Modern Approach[M]. 4th ed. Upper Saddle River: Prentice Hall, 2021: 123-145.
```

**注意事项：**
- 第1版不标注版本
- 引用整本书时不标注页码
- 出版地为城市名称

### 5.4 学位论文格式

**基本格式：**
```
[序号] 作者. 题名[D]. 保存地: 保存单位, 年.
```

**示例：**
```
[5] 赵六. 智能命令行助手系统的设计与实现[D]. 北京: 清华大学, 2023.
[6] Brown A. Design and implementation of AI-powered shell assistant[D]. Cambridge: MIT, 2023.
```

### 5.5 会议论文格式

**基本格式：**
```
[序号] 作者. 题名[C]//会议论文集名. 出版地: 出版者, 年: 起止页码.
```

**示例：**
```
[7] 孙七. 基于Transformer的命令翻译模型[C]//第十届中国计算机大会论文集. 北京: 电子工业出版社, 2023: 234-241.
```

### 5.6 网络资源格式

**基本格式：**
```
[序号] 作者. 题名[EB/OL]. (发表日期)[引用日期]. 网址.
```

**示例：**
```
[8] OpenAI. GPT-4 Technical Report[EB/OL]. (2023-03-15)[2024-01-10]. https://openai.com/research/gpt-4.
[9] Microsoft. PowerShell Documentation[EB/OL]. [2024-01-10]. https://docs.microsoft.com/powershell/.
```

**注意事项：**
- 有发表日期的标注发表日期
- 无发表日期的省略发表日期
- 引用日期必须标注
- URL应完整且可访问

### 5.7 文内引用格式

**顺序编码制：**
- 格式：[序号]
- 示例：根据文献[1]的研究表明...
- 多个文献：[1-3]或[1,3,5]
- 位置：标点符号之前

**著者-出版年制（可选）：**
- 格式：(作者, 年份)
- 示例：根据Smith等人(2023)的研究...

## 6. 术语和缩写规范

### 6.1 术语使用原则

**首次出现规则：**
- 中文术语首次出现时应提供英文对照
- 格式：中文术语（英文全称，English Abbreviation）
- 示例：自然语言处理（Natural Language Processing，NLP）

**术语一致性：**
- 同一术语在全文中使用统一翻译
- 建立术语对照表
- 避免同义词混用

**专业术语：**
- 使用行业标准术语
- 避免自造词汇
- 必要时提供定义或解释

## 8. 特殊格式规范

### 8.1 列表格式

**无序列表：**
- 使用实心圆点（•）或短横线（-）
- 缩进：2字符
- 行距：与正文相同
- 示例：
  - 第一项内容
  - 第二项内容
  - 第三项内容

**有序列表：**
- 使用阿拉伯数字编号
- 格式：1. 2. 3. 或 (1) (2) (3)
- 缩进：2字符
- 示例：
  1. 第一步操作
  2. 第二步操作
  3. 第三步操作

**多级列表：**
- 第一级：1. 2. 3.
- 第二级：(1) (2) (3)
- 第三级：① ② ③
- 第四级：a. b. c.

### 8.2 引用格式

**直接引用：**
- 使用双引号
- 标注文献来源
- 示例："人工智能是研究、开发用于模拟、延伸和扩展人的智能的理论、方法、技术及应用系统的一门新的技术科学"[1]。

**间接引用：**
- 不使用引号
- 标注文献来源
- 示例：根据Smith等人的研究[2]，自然语言处理技术在近年来取得了显著进展。

**长引用（>3行）：**
- 单独成段
- 缩进：左右各缩进2字符
- 字号：小五号
- 行距：单倍行距

### 8.3 脚注格式

**脚注编号：**
- 使用上标数字：①②③
- 位置：标点符号之后
- 编号：每页重新编号或全文连续编号

**脚注内容：**
- 字体：宋体，小五号
- 位置：页面底部
- 分隔线：3cm短线
- 示例：
  ```
  ① PowerShell是微软开发的任务自动化和配置管理框架。
  ```

### 8.4 附录格式

**附录编号：**
- 格式：附录A、附录B、附录C
- 英文：Appendix A, Appendix B

**附录标题：**
- 字体：黑体，三号，加粗，居中
- 格式：附录A 标题内容
- 示例：附录A 配置文件示例

**附录内容：**
- 字体：与正文相同
- 编号：独立编号系统
- 图表：附录A图1、附录A表1

## 10. 常见格式问题及解决方案

### 10.1 页码问题

**问题：页码不连续**
- 原因：分节符设置不当
- 解决：检查分节符，设置页码连续

**问题：页码格式不统一**
- 原因：不同节使用了不同格式
- 解决：统一设置页码格式

### 10.2 图表问题

**问题：图表编号不连续**
- 原因：手动编号或删除了中间的图表
- 解决：使用自动编号功能，重新编号

**问题：图表位置不当**
- 原因：图表浮动设置不当
- 解决：设置图表位置为"嵌入型"或"上下型环绕"

**问题：图表跨页断开**
- 原因：段落设置允许跨页
- 解决：设置图表所在段落"段落中不分页"

### 10.3 参考文献问题

**问题：文献编号不连续**
- 原因：删除了部分文献但未重新编号
- 解决：使用文献管理工具或手动重新编号

**问题：文献格式不统一**
- 原因：不同来源的文献格式不同
- 解决：统一按GB/T 7714-2015标准格式化

**问题：文内引用与文献列表不对应**
- 原因：修改时未同步更新
- 解决：逐一核对，确保对应关系正确

### 10.4 代码格式问题

**问题：代码缩进混乱**
- 原因：复制粘贴时格式丢失
- 解决：使用代码编辑器格式化后再粘贴

**问题：代码字体不统一**
- 原因：部分代码使用了不同字体
- 解决：全选代码，统一设置为Consolas字体

**问题：代码行号显示不正确**
- 原因：使用了表格或文本框
- 解决：使用专业代码编辑器生成带行号的代码图片

### 10.5 术语问题

**问题：同一术语多种翻译**
- 原因：不同章节使用了不同翻译
- 解决：建立术语表，全文查找替换

**问题：缩写未说明**
- 原因：忘记在首次出现时给出全称
- 解决：查找所有缩写，补充全称说明
