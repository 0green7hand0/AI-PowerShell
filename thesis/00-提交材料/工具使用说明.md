# 提交材料准备工具使用说明

本文档详细说明如何使用各个准备工具。

## 工具列表

### 1. generate_pdfs.py - PDF生成工具

**功能**: 将Markdown文档批量转换为PDF格式

**使用方法**:
```bash
python scripts/generate_pdfs.py
```

**前提条件**:
- 已安装Pandoc
- 已安装LaTeX (MiKTeX或TeX Live)
- 系统已安装中文字体

**输出位置**: `thesis/00-提交材料/PDF文档/`

**生成的文件**:
- 01-毕业论文.pdf
- 02-系统设计文档.pdf
- 03-用户手册.pdf
- 04-开发文档.pdf

**注意事项**:
- 首次运行可能需要下载LaTeX包
- 转换大文件可能需要几分钟
- 如果失败，查看错误信息

### 2. prepare_source_code.py - 源代码准备工具

**功能**: 整理、清理和打包源代码

**使用方法**:
```bash
python scripts/prepare_source_code.py
```

**主要操作**:
1. 复制源代码文件到临时目录
2. 排除临时文件和缓存
3. 清理调试代码
4. 检查README完整性
5. 创建压缩包
6. 创建Git标签

**输出位置**: `thesis/00-提交材料/源代码/`

**生成的文件**:
- AI-PowerShell/ (源代码目录)
- AI-PowerShell-源代码-YYYYMMDD.zip (压缩包)
- 源代码说明.md (说明文档)

**排除的文件**:
- __pycache__/
- *.pyc, *.pyo, *.pyd
- .pytest_cache/
- .coverage, htmlcov/
- .git/
- .vscode/, .idea/
- node_modules/
- *.log
- venv/, env/

**注意事项**:
- 会自动清理调试代码
- 会创建Git标签 (需要Git)
- 压缩包大小通常在10-50MB

### 3. prepare_other_materials.py - 其他材料准备工具

**功能**: 收集测试报告、演示视频、截图等材料

**使用方法**:
```bash
python scripts/prepare_other_materials.py
```

**收集的材料**:
1. 测试报告 (从 `thesis/06-测试报告/`)
2. 演示视频 (从 `thesis/04-答辩材料/videos/`)
3. 截图 (从 `thesis/07-项目资料/screenshots/`)
4. 性能数据 (从 `thesis/07-项目资料/data/`)

**输出位置**: `thesis/00-提交材料/其他材料/`

**生成的文件**:
- 测试报告.pdf
- 演示视频.mp4
- screenshots/ (截图目录)
- performance-data/ (数据目录)
- 材料检查清单.md
- 材料准备总结.md

**注意事项**:
- 需要先准备好源材料
- 会自动创建检查清单
- 会生成准备总结报告

### 4. organize_submission.py - 提交目录组织工具

**功能**: 创建标准的提交目录结构并打包

**使用方法**:
```bash
python scripts/organize_submission.py
```

**交互输入**:
- 学生姓名
- 学号

**主要操作**:
1. 创建标准目录结构
2. 从各个位置复制材料
3. 生成提交说明README
4. 创建检查清单
5. 打包为ZIP文件

**输出位置**: `thesis/00-提交材料/`

**生成的文件**:
- 毕业设计-姓名-学号/ (提交目录)
- 毕业设计-姓名-学号.zip (压缩包)
- 毕业设计-姓名-学号-检查清单.md

**目录结构**:
```
毕业设计-姓名-学号/
├── 01-毕业论文/
├── 02-设计文档/
├── 03-用户手册/
├── 04-答辩材料/
├── 05-开发文档/
├── 06-源代码/
├── 07-测试报告/
├── 08-其他材料/
└── README.md
```

**注意事项**:
- 需要先完成前3个步骤
- 会自动生成详细的README
- 压缩包大小通常在50-200MB

### 5. prepare_all_submission.py - 一键准备工具

**功能**: 自动执行所有准备步骤

**使用方法**:
```bash
python scripts/prepare_all_submission.py
```

**执行顺序**:
1. 生成PDF文档
2. 整理源代码
3. 准备其他材料
4. 组织提交目录

**输出**:
- 显示每个步骤的执行结果
- 生成总结报告
- 标记成功/失败的步骤

**注意事项**:
- 需要确认后才开始执行
- 如果某步骤失败，会继续执行后续步骤
- 最后会显示总体结果

## 使用流程

### 推荐流程

**第一次使用**:
1. 先分别运行各个工具，了解每个工具的功能
2. 检查每个步骤的输出
3. 确认所有材料都已准备好

**后续使用**:
1. 直接运行一键准备工具
2. 检查输出结果
3. 验证最终压缩包

### 详细流程

#### 步骤1: 准备源材料

确保以下材料已准备:
- [ ] 毕业论文 (`thesis/01-论文/thesis.md`)
- [ ] 设计文档 (`thesis/02-设计文档/design-doc.md`)
- [ ] 用户手册 (`thesis/03-用户手册/user-manual.md`)
- [ ] 答辩PPT (`thesis/04-答辩材料/`)
- [ ] 开发文档 (`thesis/05-开发文档/developer-guide.md`)
- [ ] 测试报告 (`thesis/06-测试报告/`)
- [ ] 演示视频 (`thesis/04-答辩材料/videos/`)
- [ ] 截图 (`thesis/07-项目资料/screenshots/`)

#### 步骤2: 安装依赖

**Windows**:
```powershell
# 安装Pandoc
choco install pandoc

# 安装MiKTeX
choco install miktex
```

**Linux**:
```bash
# Ubuntu/Debian
sudo apt-get install pandoc texlive-xetex texlive-lang-chinese
```

**macOS**:
```bash
brew install pandoc
brew install --cask mactex
```

#### 步骤3: 运行工具

**方法A: 一键准备**
```bash
python scripts/prepare_all_submission.py
```

**方法B: 分步执行**
```bash
# 1. 生成PDF
python scripts/generate_pdfs.py

# 2. 整理源代码
python scripts/prepare_source_code.py

# 3. 准备其他材料
python scripts/prepare_other_materials.py

# 4. 组织提交目录
python scripts/organize_submission.py
```

#### 步骤4: 检查结果

1. 查看生成的文件
2. 打开PDF检查质量
3. 解压压缩包测试
4. 运行源代码验证
5. 播放演示视频

#### 步骤5: 最终提交

1. 使用检查清单逐项检查
2. 创建多份备份
3. 按要求提交

## 故障排除

### PDF生成失败

**问题**: Pandoc未安装
**解决**: 安装Pandoc和LaTeX

**问题**: 中文乱码
**解决**: 确保安装了中文字体，使用XeLaTeX引擎

**问题**: 转换超时
**解决**: 分章节转换，或简化文档

### 源代码打包失败

**问题**: 文件过大
**解决**: 检查是否包含了不必要的文件

**问题**: Git标签创建失败
**解决**: 确保在Git仓库中，或跳过此步骤

### 材料收集失败

**问题**: 找不到文件
**解决**: 确保文件在正确的位置

**问题**: 视频文件过大
**解决**: 压缩视频文件

### 提交目录组织失败

**问题**: 缺少必需文件
**解决**: 先完成前面的步骤

**问题**: 压缩包过大
**解决**: 压缩图片和视频

## 高级用法

### 自定义PDF样式

编辑 `scripts/generate_pdfs.py`，修改Pandoc参数:

```python
cmd = [
    "pandoc",
    str(input_file),
    "-o", str(output_file),
    "--pdf-engine=xelatex",
    "-V", "CJKmainfont=SimSun",  # 修改字体
    "-V", "geometry:margin=2.54cm",  # 修改页边距
    "-V", "fontsize=12pt",  # 修改字号
    "--toc",
    "--toc-depth=3",
]
```

### 自定义文件过滤

编辑 `scripts/prepare_source_code.py`，修改排除规则:

```python
self.exclude_patterns = {
    '__pycache__',
    '*.pyc',
    # 添加更多规则
}
```

### 自定义目录结构

编辑 `scripts/organize_submission.py`，修改目录列表:

```python
directories = [
    "01-毕业论文",
    "02-设计文档",
    # 添加或修改目录
]
```

## 常见问题

### Q1: 工具运行失败怎么办?
**A**: 查看错误信息，检查依赖是否安装，参考故障排除部分

### Q2: 可以手动准备吗?
**A**: 可以，参考各子目录的README文档

### Q3: 压缩包太大怎么办?
**A**: 压缩图片和视频，移除不必要的文件

### Q4: 如何修改学生信息?
**A**: 运行organize_submission.py时会提示输入

### Q5: 可以重复运行工具吗?
**A**: 可以，工具会覆盖之前的输出
