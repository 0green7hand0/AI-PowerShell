# 提交材料准备 - 快速开始指南

本指南帮助你快速准备毕业设计的所有提交材料。

## 🚀 一键准备 (推荐)

如果所有材料都已准备好，使用一键脚本:

```bash
python scripts/prepare_all_submission.py
```

该脚本会自动完成所有步骤，大约需要5-10分钟。

## 📋 准备流程

### 前提条件

确保已完成:
- ✅ 毕业论文已编写完成
- ✅ 系统设计文档已完成
- ✅ 用户手册已完成
- ✅ 答辩PPT已制作
- ✅ 开发文档已完成
- ✅ 测试报告已编写
- ✅ 演示视频已录制

### 步骤1: 生成PDF文档 (30分钟)

将所有Markdown文档转换为PDF:

```bash
python scripts/generate_pdfs.py
```

**需要安装**:
- Pandoc: `choco install pandoc` (Windows)
- MiKTeX: `choco install miktex` (Windows)

**手动方法**: 使用Typora或在线工具转换

### 步骤2: 整理源代码 (1小时)

清理和打包源代码:

```bash
python scripts/prepare_source_code.py
```

**自动完成**:
- 复制源代码文件
- 排除临时文件
- 清理调试代码
- 创建压缩包
- 创建Git标签

### 步骤3: 准备其他材料 (2-4小时)

收集测试报告、视频等:

```bash
python scripts/prepare_other_materials.py
```

**需要准备**:
- 测试报告 (放在 `thesis/06-测试报告/`)
- 演示视频 (放在 `thesis/04-答辩材料/videos/`)
- 截图 (放在 `thesis/07-项目资料/screenshots/`)
- 性能数据 (放在 `thesis/07-项目资料/data/`)

### 步骤4: 组织提交目录 (30分钟)

创建最终提交目录:

```bash
python scripts/organize_submission.py
```

**输入信息**:
- 学生姓名
- 学号

**自动完成**:
- 创建标准目录结构
- 复制所有材料
- 生成README
- 创建压缩包

## ⚠️ 常见问题

### Q: PDF生成失败?
**A**: 安装Pandoc和LaTeX，或使用Typora手动转换

### Q: 压缩包太大?
**A**: 压缩图片和视频，或移除不必要的文件

### Q: 源代码无法运行?
**A**: 检查依赖安装和配置文件

### Q: 材料不完整?
**A**: 查看检查清单，补充缺失的材料
