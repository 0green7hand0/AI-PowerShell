# 最终定稿检查清单

**创建日期**: 2025-11-11  
**用途**: 确保文档达到提交标准

---

## 使用说明

本检查清单用于最终定稿前的全面检查。请逐项检查，确保所有项目都符合要求。

**检查方法**:
- ✅ 表示已检查且符合要求
- ❌ 表示不符合要求，需要修改
- ⏭️ 表示不适用或可选

**检查顺序**:
1. 先检查高优先级项目
2. 再检查中优先级项目
3. 最后检查低优先级项目

---

## 二、格式规范性检查 (高优先级)

### 2.1 页面设置

- [ ] 纸张大小: A4
- [ ] 页边距: 上下2.54cm，左右3.17cm（或按学校要求）
- [ ] 行距: 1.5倍（或按学校要求）
- [ ] 页码位置正确
- [ ] 页眉页脚设置正确（如需要）

### 2.2 字体字号

- [ ] 中文字体: 宋体（正文）
- [ ] 英文字体: Times New Roman（正文）
- [ ] 正文字号: 小四
- [ ] 一级标题: 黑体三号加粗
- [ ] 二级标题: 黑体四号加粗
- [ ] 三级标题: 黑体小四加粗
- [ ] 图表标题: 五号
- [ ] 图表内容: 五号
- [ ] 代码字体: Consolas或Courier New
- [ ] 代码字号: 小五

### 2.3 标题格式

- [ ] 标题层次清晰（最多三级）
- [ ] 标题编号正确（1, 1.1, 1.1.1）
- [ ] 标题与正文之间有适当间距
- [ ] 标题不跨页
- [ ] 标题后不加标点符号

### 2.4 图表格式

- [ ] 图题在图下方，居中
- [ ] 表题在表上方，居中
- [ ] 图表编号格式: 图X-X, 表X-X
- [ ] 图表编号连续无跳号
- [ ] 图表清晰可辨
- [ ] 图表大小适中
- [ ] 图表位置合理（尽量在引用处附近）
- [ ] 表格线条规范（三线表）
- [ ] 表格无空单元格（或用"-"表示）

### 2.5 代码格式

- [ ] 代码块有语言标识
- [ ] 代码缩进正确
- [ ] 代码行长度适中（建议≤80字符）
- [ ] 代码有必要的注释
- [ ] 代码语法正确
- [ ] 代码格式统一

### 2.6 参考文献格式

- [ ] 参考文献格式符合GB/T 7714-2015标准
- [ ] 参考文献格式统一
- [ ] 作者姓名格式正确
- [ ] 期刊名、书名格式正确
- [ ] 出版信息完整
- [ ] 页码范围正确
- [ ] 网址和访问日期正确（网络资源）

## 附录: 常见问题快速检查

### A1. 图表编号快速检查

**方法**: 使用查找功能搜索"图"和"表"，检查编号是否连续

**命令**:
```
# 在文档中搜索
图\d+-\d+
表\d+-\d+
```

### A2. 参考文献快速检查

**方法**: 检查引用编号是否连续，是否都有对应文献

**步骤**:
1. 搜索所有 [数字] 格式的引用
2. 检查编号是否连续
3. 对照参考文献列表

### A3. 数据一致性快速检查

**方法**: 搜索关键性能指标，检查数值是否一致

**关键词**:
- 翻译准确率
- 响应时间
- 内存占用
- 测试覆盖率
- 缓存命中率

### A4. 代码语法快速检查

**方法**: 将代码复制到IDE中检查语法

**工具**:
- Python: PyCharm, VS Code
- PowerShell: VS Code, PowerShell ISE
- 在线工具: repl.it, jsfiddle等
