**创建日期**: 2025-11-11  
**用途**: 指导如何收集和整理审阅反馈

---

## 一、指导教师反馈收集

### 1.1 提交前准备

**检查清单**:
- [ ] 所有文档已完成自我审阅
- [ ] 已知问题已在审阅材料包中说明
- [ ] 文档格式基本规范
- [ ] 准备好纸质版或电子版材料
- [ ] 预约好审阅时间

### 1.2 提交方式

**方式一: 面对面提交**
- 优点: 可以当面讨论，获得即时反馈
- 准备: 打印主要文档，准备笔记本记录
- 流程:
  1. 简要介绍文档结构和主要内容
  2. 说明已知问题和疑问
  3. 请教师审阅并记录反馈
  4. 确认修改要点和时间安排

**方式二: 电子邮件提交**
- 优点: 教师可以详细审阅，留下书面反馈
- 准备: 整理好所有电子文档
- 邮件模板:

```
主题: 毕业设计文档审阅请求 - [学生姓名]

[教师姓名]老师，您好！

我已完成毕业设计的主要文档编写工作，现提交给您审阅。

【文档清单】
1. 毕业论文主文档 (约40,000字)
2. 系统设计文档 (约15,000字)
3. 用户手册 (约10,000字)
4. 答辩材料
5. 开发文档 (约12,000字)

【审阅重点】
1. 研究内容的完整性和创新性
2. 技术方案的合理性
3. 实验数据的可靠性
4. 论文结构和语言表达

【已知问题】
1. 部分图表编号需要调整
2. 性能数据需要统一
3. [其他问题]

【附件】
- advisor-review-package.md (审阅材料包)
- [其他文档]

期望反馈时间: [日期]

如有任何问题，请随时联系我。

学生: [姓名]
学号: [学号]
联系方式: [电话/邮箱]
日期: 2025-11-11
```

## 二、同学互审反馈收集

### 2.1 互审准备

**选择互审同学**:
- 建议: 2-3名同学
- 标准: 
  - 同专业或相关专业
  - 认真负责
  - 有一定技术基础

**准备材料**:
- [ ] 论文主文档
- [ ] 互审反馈表
- [ ] 互审指南

### 2.2 互审指南

**提供给互审同学的指南**:

```markdown
# 毕业设计互审指南

感谢你帮助审阅我的毕业设计文档！

## 审阅重点

### 1. 可读性
- 内容是否容易理解？
- 逻辑是否清晰？
- 是否有表述不清的地方？

### 2. 完整性
- 是否有遗漏的内容？
- 图表是否完整？
- 参考文献是否充分？

### 3. 技术性
- 技术方案是否合理？
- 代码示例是否正确？
- 测试数据是否可信？

### 4. 格式
- 格式是否统一？
- 图表编号是否正确？
- 标点符号是否规范？

## 反馈方式

请在文档中直接标注问题，或填写反馈表。

## 时间安排

- 审阅时间: 3-5天
- 反馈截止: [日期]

再次感谢！
```

## 三、反馈响应流程

### 3.1 反馈分类

**按来源分类**:
- 指导教师反馈 (最高优先级)
- 同学互审反馈 (参考)
- 自我审阅发现 (基础)

**按类型分类**:
- 内容问题
- 格式问题
- 语言问题
- 技术问题

**按优先级分类**:
- 必须修改 (影响答辩)
- 重要修改 (影响质量)
- 建议修改 (锦上添花)

### 3.2 修改计划制定

**步骤**:
1. 整理所有反馈
2. 去重和合并相似问题
3. 评估修改难度和时间
4. 制定修改顺序
5. 设定完成时间

**修改计划模板**:

```markdown
# 基于反馈的修改计划

**制定日期**: ___________
**计划完成日期**: ___________

## 第一批修改 (必须修改)

| 序号 | 问题 | 来源 | 预计时间 | 负责人 | 截止日期 |
|------|------|------|----------|--------|----------|
| 1 | [问题1] | 指导教师 | 2小时 | [姓名] | [日期] |
| 2 | [问题2] | 指导教师 | 1小时 | [姓名] | [日期] |

## 第二批修改 (重要修改)

[同上格式]

## 第三批修改 (建议修改)

[同上格式]

## 时间安排

- 第一批: [开始日期] - [结束日期]
- 第二批: [开始日期] - [结束日期]
- 第三批: [开始日期] - [结束日期]
- 总计: [总天数]
```

### 3.3 修改执行

**执行原则**:
1. 严格按照指导教师意见修改
2. 优先处理高优先级问题
3. 保持文档一致性
4. 及时更新修改跟踪表
5. 完成后自我验证

**修改记录**:
- 在modification-tracking.md中更新状态
- 记录修改内容和时间
- 标注修改人

### 3.4 修改验证

**验证清单**:
- [ ] 所有必须修改项已完成
- [ ] 修改后文档一致性良好
- [ ] 没有引入新的错误
- [ ] 格式规范统一
- [ ] 数据准确一致

### 3.5 再次提交

**提交前检查**:
- [ ] 所有反馈已响应
- [ ] 修改记录完整
- [ ] 文档质量提升明显
- [ ] 准备好修改说明

## 五、注意事项

### 5.1 沟通技巧

**与指导教师沟通**:
- 保持谦虚态度
- 认真记录意见
- 不懂就问
- 及时反馈进度

**与同学沟通**:
- 表达感谢
- 认真对待意见
- 互相学习
- 保持联系

### 5.2 时间管理

**合理安排时间**:
- 留出充足的修改时间
- 考虑多轮审阅的可能
- 预留缓冲时间
- 不要拖到最后

### 5.3 质量控制

**确保修改质量**:
- 理解反馈的本质
- 不要敷衍修改
- 保持文档一致性
- 修改后自我验证

## 六、常见问题

### Q1: 如果指导教师和同学的意见不一致怎么办？

**A**: 以指导教师的意见为准。同学的意见可以作为参考，但最终决策权在指导教师。

### Q2: 如果反馈意见太多，时间不够怎么办？

**A**: 
1. 优先处理指导教师的必须修改项
2. 与教师沟通，说明情况
3. 请求延长时间或分批修改
4. 合理安排优先级

### Q3: 如果不理解某个反馈意见怎么办？

**A**:
1. 及时向反馈人请教
2. 查阅相关资料
3. 与同学讨论
4. 不要猜测，确保理解正确

### Q4: 修改后需要再次审阅吗？

**A**: 
- 重大修改: 必须再次审阅
- 一般修改: 建议再次审阅
- 小修改: 可以自我验证后提交
