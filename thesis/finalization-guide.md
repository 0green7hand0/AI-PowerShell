## 一、定稿流程概述

### 1.1 定稿阶段

```
自我审阅 → 第一轮修改 → 指导教师审阅 → 第二轮修改 → 
同学互审 → 第三轮修改 → 最终检查 → 定稿 → 提交
```

### 1.2 时间安排

| 阶段 | 预计时间 | 累计时间 |
|------|----------|----------|
| 自我审阅 | 1天 | 1天 |
| 第一轮修改 | 2-3天 | 3-4天 |
| 指导教师审阅 | 3-5天 | 6-9天 |
| 第二轮修改 | 2-3天 | 8-12天 |
| 同学互审 | 3-5天 | 11-17天 |
| 第三轮修改 | 1-2天 | 12-19天 |
| 最终检查 | 1天 | 13-20天 |
| 定稿提交 | 1天 | 14-21天 |

**建议**: 预留至少3周时间完成定稿工作

## 三、常见问题修复

### 3.1 图表编号问题

**问题**: 图表编号不连续

**修复步骤**:
1. 列出所有图表及其编号
2. 重新编号（确保连续）
3. 更新图目录和表目录
4. 更新正文中的所有引用

**工具**:
```python
# 使用Python脚本批量检查
import re

def check_figure_numbers(file_path):
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()
    
    figures = re.findall(r'图(\d+)-(\d+)', content)
    for i, (chapter, num) in enumerate(figures, 1):
        print(f"图{chapter}-{num}")
```

### 3.2 数据不一致问题

**问题**: 不同位置的数据不一致

**修复步骤**:
1. 确定最终数据（使用最新测试结果）
2. 搜索所有数据出现位置
3. 统一替换为最终数据
4. 验证摘要、正文、图表中的数据一致

**建议的统一数据**:
```markdown
- 翻译准确率: 92%
- 平均响应时间: 1.5秒
- 内存占用: 380MB (不含AI模型)
- 测试覆盖率: 90%
- 缓存命中率: 65%
```

### 3.3 代码语法错误

**问题**: 代码存在语法错误

**修复步骤**:
1. 将代码复制到IDE中
2. 运行语法检查
3. 修复所有错误
4. 更新文档中的代码

**验证方法**:
```bash
# Python代码验证
python -m py_compile your_code.py

# PowerShell代码验证
pwsh -Command "Test-Path your_script.ps1"
```

### 3.4 参考文献格式问题

**问题**: 参考文献格式不统一

**修复步骤**:
1. 确定使用的格式标准（GB/T 7714-2015）
2. 逐条检查参考文献
3. 统一格式
4. 验证引用编号

**格式模板**:
```markdown
期刊论文:
[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.

专著:
[序号] 作者. 书名[M]. 版本. 出版地: 出版者, 年: 起止页码.

学位论文:
[序号] 作者. 题名[D]. 保存地: 保存单位, 年.

网络资源:
[序号] 作者. 题名[EB/OL]. (发表日期)[引用日期]. 网址.
```

### 3.5 标点符号问题

**问题**: 中英文标点混用

**修复步骤**:
1. 使用查找替换功能
2. 将英文括号()替换为中文括号（）
3. 检查其他标点符号
4. 验证修改结果

**批量替换**:
```
查找: \(([^)]+)\)
替换: （$1）
```

## 四、生成最终版本

### 4.1 版本命名规范

**命名格式**:
```
[学号]_[姓名]_毕业论文_v[版本号]_[日期].pdf

示例:
20190001_张三_毕业论文_v3.0_20251111.pdf
```

**版本说明**:
- v1.0: 初稿
- v2.0: 第一次修改后
- v3.0: 第二次修改后
- vFinal: 最终定稿版

### 4.2 生成PDF

**方法一: Word转PDF**
1. 打开Word文档
2. 文件 → 另存为 → PDF
3. 选择"标准"质量
4. 确保"文档结构标记"已勾选
5. 保存

**方法二: Markdown转PDF**
1. 使用Typora或Pandoc
2. 导出为PDF
3. 检查格式是否正确
4. 必要时调整样式

**质量检查**:
- [ ] PDF可以正常打开
- [ ] 所有内容完整
- [ ] 图表清晰
- [ ] 链接有效（如有）
- [ ] 文件大小合理（建议<50MB）

### 4.3 生成Word版本

**如果需要提交Word版本**:

1. 确保使用的是最新版本
2. 检查兼容性
3. 嵌入所有字体（可选）
4. 保存为.docx格式

**注意事项**:
- 不同版本的Word可能显示不同
- 建议使用PDF作为最终提交格式
- Word版本仅作为备份

### 4.4 文件组织

**提交文件夹结构**:
```
毕业设计-张三-20190001/
├── 01-论文/
│   ├── 20190001_张三_毕业论文_vFinal_20251111.pdf
│   └── 20190001_张三_毕业论文_vFinal_20251111.docx (可选)
├── 02-设计文档/
│   └── 系统设计文档.pdf
├── 03-用户手册/
│   └── 用户手册.pdf
├── 04-答辩材料/
│   ├── 答辩PPT.pdf
│   ├── 答辩PPT.pptx
│   └── 演示视频.mp4 (可选)
├── 05-源代码/

└── README.md
```

## 八、常见问题

### Q1: 定稿后发现错误怎么办？

**A**: 
1. 立即联系指导教师
2. 说明错误情况
3. 请求修改机会
4. 如果允许，尽快修改并重新提交

### Q2: 提交截止日期快到了，还没完成怎么办？

**A**:
1. 优先完成核心内容
2. 确保基本要求达标
3. 与指导教师沟通
4. 请求延期（如可能）

### Q3: 不同版本的文档内容不一致怎么办？

**A**:
1. 确定最终版本
2. 删除或归档旧版本
3. 只提交最终版本
4. 做好版本标记

### Q4: 文件太大无法上传怎么办？

**A**:
1. 压缩图片质量
2. 删除不必要的内容
3. 分卷压缩
4. 使用网盘分享链接
