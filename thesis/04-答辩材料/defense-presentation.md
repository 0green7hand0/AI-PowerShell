## 幻灯片3：研究背景

### 命令行工具的重要性与挑战

**现状：**
- PowerShell是Windows系统管理的核心工具
- 功能强大但学习曲线陡峭
- 复杂的语法和大量的命令集

**问题：**
- 初学者难以快速上手
- 非专业用户操作门槛高
- 容易因误操作导致系统损坏

**机遇：**
- AI技术特别是大语言模型快速发展
- 自然语言处理技术日趋成熟
- 本地AI模型部署成为可能

## 幻灯片8：核心模块设计 - AI引擎

### 混合翻译策略

**翻译流程：**

```
用户输入
  ↓
检查缓存 ──→ 命中 ──→ 返回结果
  ↓ 未命中
规则匹配 ──→ 匹配 ──→ 返回结果
  ↓ 未匹配
AI模型生成 ──→ 生成命令
  ↓
错误检测与修正
  ↓
缓存结果 ──→ 返回结果
```

**关键技术：**
- **规则匹配**：正则表达式，快速响应（<5ms）
- **AI模型**：LLaMA/Ollama，准确翻译（~1.5s）
- **LRU缓存**：提高命中率（65%）
- **错误检测**：语法检查，参数验证

## 幻灯片9：核心模块设计 - 安全引擎

### 三层安全保护机制

**第一层：命令白名单验证**
- 30+种危险命令模式识别
- 风险等级评估（SAFE → CRITICAL）
- 正则表达式模式匹配

**第二层：动态权限检查**
- 管理员命令自动识别
- 当前权限状态检测
- 用户确认流程

**第三层：沙箱隔离执行**
- Docker容器隔离
- 资源限制（CPU、内存、网络）
- 可选启用，平衡安全与性能

**风险等级定义：**
- 🟢 SAFE：安全命令，直接执行
- 🟡 LOW：低风险，提示用户
- 🟠 MEDIUM：中风险，需要确认
- 🔴 HIGH：高风险，强制确认
- ⛔ CRITICAL：极高风险，拒绝执行

## 幻灯片14：关键技术实现

### 中文编码问题解决

**问题：** PowerShell输出中文乱码

**解决方案：**
```python
# 1. 设置正确的编码
process = subprocess.Popen(
    ['pwsh', '-Command', command],
    stdout=subprocess.PIPE,
    stderr=subprocess.PIPE,
    encoding='utf-8'  # 指定UTF-8编码
)

# 2. Windows平台特殊处理
if platform.system() == 'Windows':
    # 设置代码页为UTF-8
    subprocess.run(['chcp', '65001'], 
                   capture_output=True)
```

### 跨平台路径处理

**问题：** Windows和Unix路径分隔符不同

**解决方案：**
```python
from pathlib import Path

# 使用pathlib统一处理路径
config_path = Path.home() / '.ai-powershell' / 'config.yaml'
# 自动适配平台：
# Windows: C:\Users\xxx\.ai-powershell\config.yaml
# Linux: /home/xxx/.ai-powershell/config.yaml
```

## PPT制作说明

### 设计风格建议

**配色方案：**
- 主色调：深蓝色 (#1E3A8A) - 专业、技术感
- 辅助色：浅蓝色 (#3B82F6) - 清新、现代
- 强调色：橙色 (#F59E0B) - 突出重点
- 背景色：白色/浅灰 (#F9FAFB) - 简洁清晰

**字体建议：**
- 标题：微软雅黑 Bold, 32-44pt
- 正文：微软雅黑, 18-24pt
- 代码：Consolas/Courier New, 14-18pt
- 英文：Arial/Helvetica

**布局原则：**
- 每页内容不超过7行
- 使用项目符号和编号
- 适当留白，避免拥挤
- 重点内容使用颜色或图标突出

**动画效果：**
- 标题：淡入效果
- 内容：逐条出现
- 图表：擦除或缩放效果
- 过渡：平滑切换，避免过于花哨

### 图表制作建议

**架构图：**
- 使用draw.io或Visio制作
- 清晰的层次结构
- 统一的图标风格
- 适当的颜色区分

**数据图表：**
- 使用Excel或Python matplotlib
- 柱状图、折线图、饼图
- 清晰的标签和图例
- 数据可视化要直观

**代码片段：**
- 语法高亮
- 关键部分加粗或标注
- 添加注释说明
- 控制代码行数（<15行）

### 演示技巧

**时间控制：**
- 总时长：10-15分钟
- 背景部分：2-3分钟
- 设计部分：3-4分钟
- 实现部分：3-4分钟
- 测试部分：2-3分钟
- 总结部分：1-2分钟

**演讲要点：**
- 开场简洁有力
- 突出创新点和亮点
- 用数据说话
- 准备演示视频或动图
- 预留提问时间

**注意事项：**
- 提前熟悉内容
- 准备备用U盘
- 检查设备兼容性
- 控制语速，清晰表达
- 保持自信和微笑
