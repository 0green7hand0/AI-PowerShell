### 场景3：安全机制演示（2分钟）

#### 子场景3.1：危险命令拦截（1分钟）

**[01:00-01:05] 输入危险命令**
```
>>> 删除所有文件
```
**讲解词**："现在演示安全机制。我输入一个危险操作：'删除所有文件'。"

**[01:05-01:30] 展示安全拦截**
```
[AI] 翻译结果: Remove-Item * -Recurse -Force
[安全] ⛔ 危险命令检测！
[安全] 风险等级: CRITICAL
[安全] 风险说明: 递归强制删除所有文件，可能导致数据丢失
[安全] 此命令已被拒绝执行
```
**讲解词**："系统立即识别出这是危险命令，风险等级CRITICAL，并拒绝执行。这是三层安全机制的第一层：命令白名单验证。"

**[01:30-01:40] 展示建议**
```
[建议] 如果确实需要删除文件，请：
  1. 明确指定要删除的文件或目录
  2. 先使用 Get-ChildItem 查看要删除的内容
  3. 考虑使用回收站而不是永久删除
```
**讲解词**："系统不仅拦截，还提供了安全建议，这体现了人性化设计。"

#### 子场景3.2：权限检查（1分钟）

**[01:40-01:45] 输入需要权限的命令**
```
>>> 重启计算机
```
**讲解词**："接下来演示权限检查。我输入'重启计算机'。"

**[01:45-02:00] 展示权限检查**
```
[AI] 翻译结果: Restart-Computer
[安全] ⚠️ 权限检查
[安全] 此命令需要管理员权限
[安全] 当前权限: 普通用户
[安全] 风险等级: HIGH
[提示] 是否以管理员身份执行? (y/n):
```
**讲解词**："系统检测到这个命令需要管理员权限，当前是普通用户，风险等级HIGH。这是第二层安全机制：动态权限检查。"

**[02:00-02:10] 用户确认**
```
n
[取消] 用户取消执行
```
**讲解词**："我选择取消执行。系统给予用户完全的控制权，这保证了安全性和可用性的平衡。"

### 核心展示内容（必须包含）
1. ✅ **中文自然语言支持**：用户可以用中文输入，无需记忆命令

3. ✅ **三层安全机制**：危险命令拦截、权限检查、用户确认

1. 💡 **混合翻译策略**：规则匹配 + AI模型，兼顾速度和准确性

3. 💡 **风险等级分类**：SAFE、LOW、MEDIUM、HIGH、CRITICAL五个等级
4. 💡 **友好的用户体验**：清晰的提示、详细的解释、有用的建议

### 创新点（答辩时强调）
1. 🌟 **本地AI处理**：保护用户隐私，无需云端服务
2. 🌟 **完整中文支持**：全流程中文化，降低使用门槛
3. 🌟 **三层安全架构**：创新的安全设计，全面防护
4. 🌟 **模块化设计**：高内聚低耦合，易于维护和扩展

## 应急预案

### 系统无法启动
**方案A**：使用备用视频
- 提前录制好完整演示视频
- 视频时长3-4分钟
- 包含所有核心场景

**方案B**：使用截图讲解
- 准备每个场景的关键截图
- 按照脚本逐张展示
- 口头讲解演示过程

### 命令执行失败
**方案A**：重新执行
- 保持冷静，说明"我们再试一次"
- 如果再次失败，跳过该场景

**方案B**：使用预录结果
- 提前准备好执行结果的截图
- 说明"这是预期的执行结果"

### 时间不够
**优先级排序**：
1. 🔴 必须演示：场景1、场景2、场景3
2. 🟡 推荐演示：场景4
3. 🟢 可选演示：场景5、场景6

**压缩策略**：
- 减少讲解，只展示结果
- 跳过可选场景
- 加快操作速度

### 投影问题
**方案A**：调整分辨率
- 降低分辨率到1024x768
- 增大字体到24pt

**方案B**：使用笔记本屏幕
- 请评委靠近观看
- 或者使用备用视频

## 演示后的问题应对

### 可能的问题
1. **Q：演示是真实运行还是录制的？**
   - A：这是真实运行的系统，如果老师有兴趣，我可以现场输入其他命令测试。

2. **Q：如果AI翻译错误怎么办？**
   - A：系统有置信度评分，低于阈值会提示用户。用户可以选择不执行，或者手动修改命令。

3. **Q：安全机制会不会误拦截正常命令？**
   - A：我们的测试显示误报率小于5%。而且用户可以自定义安全规则，调整拦截策略。

4. **Q：系统性能如何？**
