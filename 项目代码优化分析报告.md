## 🎯 发现的问题和优化建议

### 1. 冗余模块 - cleanup 目录

**问题**: `src/cleanup/` 模块似乎是临时工具，不属于核心功能

**文件列表**:
- `src/cleanup/__init__.py`
- `src/cleanup/cli.py`
- `src/cleanup/file_analyzer.py`

**分析**:
- 这是一个文件分析工具，用于项目清理
- 不是应用的核心功能
- 应该移到 `scripts/` 或 `tools/` 目录

**建议**: 
```
选项 1: 删除（如果已完成清理任务）
选项 2: 移动到 scripts/cleanup/ 作为维护工具
选项 3: 移动到独立的 tools/ 目录
```

### 2. UI 配置管理重复

**问题**: UI 配置管理存在功能重叠

**相关文件**:
- `src/ui/config_loader.py` - 配置加载
- `src/ui/config_manager.py` - 配置管理
- `src/ui/compatibility.py` - 兼容性处理

**分析**:
- `config_loader.py` 和 `config_manager.py` 功能有重叠
- `config_manager.py` 内部调用 `config_loader.py`
- 可以合并为单一的配置管理类

**建议**:
```python
# 合并后的结构
src/ui/
├── config.py  # 统一的配置管理（合并 loader + manager）
├── compatibility.py  # 保持独立
└── ...
```

**优化方案**:
- 将 `UIConfigLoader` 作为 `UIConfigManager` 的私有方法
- 减少一个文件，简化导入路径
